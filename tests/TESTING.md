## 测试用例

### 第一阶段测试用例

#### TC-001: 文件上传 - 正常流程

**前置条件**: 无

**步骤**:
1. 准备一个 10MB 的 MP4 文件
2. 调用 `POST /api/v1/upload`
3. 验证响应

**预期结果**:
- 返回 200 状态码
- 响应包含 fileId
- 文件保存到 `uploads/videos/` 目录

**优先级**: P0

---

#### TC-002: 文件上传 - 格式不支持

**前置条件**: 无

**步骤**:
1. 准备一个 TXT 文件
2. 调用 `POST /api/v1/upload`

**预期结果**:
- 返回错误码 1001
- 提示"文件格式不支持"

**优先级**: P0

---

#### TC-003: 文件上传 - 文件超限

**前置条件**: 无

**步骤**:
1. 准备一个 3GB 的视频文件
2. 调用 `POST /api/v1/upload`

**预期结果**:
- 返回错误码 1002
- 提示"文件大小超限"

**优先级**: P0

---

#### TC-004: 视频拼接 - 正常流程

**前置条件**: 已上传 3 个视频文件

**步骤**:
1. 调用 `POST /api/v1/video/concat`
2. 获取 taskId
3. 轮询 `GET /api/v1/tasks/:taskId`
4. 验证输出文件

**预期结果**:
- 任务状态变为 completed
- 输出文件存在
- 视频时长 = 3 个视频时长之和

**优先级**: P0

---

#### TC-005: AI 配音生成 - 正常流程

**前置条件**: 无

**步骤**:
1. 准备配音文本
2. 调用 `POST /api/v1/audio/voiceover`
3. 验证输出音频

**预期结果**:
- 返回 audioId
- 音频文件存在
- 音频时长与文本长度匹配

**优先级**: P0

---

#### TC-006: 添加背景音乐 - 正常流程

**前置条件**: 已上传视频和音频

**步骤**:
1. 调用 `POST /api/v1/video/add-music`
2. 获取 taskId
3. 轮询任务状态
4. 验证输出文件

**预期结果**:
- 任务完成
- 输出视频包含背景音乐
- 背景音乐音量正确

**优先级**: P0

---

#### TC-007: 任务并发 - 压力测试

**前置条件**: 准备 20 个视频文件

**步骤**:
1. 同时提交 20 个视频处理任务
2. 监控系统资源
3. 验证所有任务完成

**预期结果**:
- 所有任务完成
- 系统未崩溃
- 平均响应时间 < 5 秒

**优先级**: P1

## 性能测试

### 性能指标

| 指标 | 目标 | 测试方法 |
|------|------|----------|
| API 响应时间 | ≤500ms | JMeter |
| 视频处理速度 | ≥0.5x 实时 | 实际测试 |
| 并发能力 | 20 路 | 压力测试 |
| 内存占用 | ≤4GB | 监控 |

### 性能测试场景

#### 场景 1: 单视频处理

- 输入：1 分钟 1080P 视频
- 操作：添加背景音乐
- 预期：处理时间 ≤2 分钟

#### 场景 2: 并发处理

- 输入：20 个 1 分钟视频
- 操作：同时提交处理
- 预期：全部在 5 分钟内完成

### 测试脚本示例

```python
# tests/test_upload.py
import requests

def test_upload_video():
    files = {'file': open('test.mp4', 'rb')}
    data = {'type': 'video'}
    response = requests.post('http://localhost:15321/api/v1/upload', files=files, data=data)
    assert response.status_code == 200
    assert 'fileId' in response.json()['data']
```
